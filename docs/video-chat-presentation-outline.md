## Slide 3 — 模組總覽

本系統包含兩大即時互動模組：

| 模組 | 用途 |
|------|------|
| 🎥 **視訊教室** | 支援一對一即時視訊上課，包含螢幕分享與教室內聊天 |
| 💬 **聊天室** | 支援課前課後文字與多媒體溝通，含獨立訊息中心 |

- 學生與老師各有專屬介面
- 兩模組共用相同的即時通訊基礎建設（WebSocket）

## Slide 4 — 視訊教室：使用者體驗流程

```
  ① 我的課程        ② 設備預檢         ③ 等待對方
  點擊「進入教室」 → 預覽攝影機/麥克風 → 系統建立連線
                                          │
  ⑤ 課程結束        ④ 上課中           ◄┘
  顯示結束畫面   ← 視訊 + 聊天 + 螢幕分享
  返回課程頁
```

**重點說明：**
- 進入前可確認設備是否正常，避免上課時才發現問題
- 攝影機無法開啟時，自動切換為純音訊模式
- 上課期間隨時可切換麥克風、攝影機、螢幕分享

## Slide 5 — 視訊教室：核心功能

| 功能 | 說明 |
|------|------|
| ✅ 即時一對一視訊通話 | 學生與老師面對面視訊上課 |
| ✅ 麥克風 / 攝影機切換 | 上課中可隨時開關 |
| ✅ 螢幕分享 | 展示教材或操作示範 |
| ✅ 教室內即時聊天 | 上課時可同步文字溝通與傳檔 |
| ✅ 房間狀態顯示 | 等待中 → 上課中 → 已結束 |
| ✅ 裝置預檢 | 進入教室前確認攝影機和麥克風 |
| ✅ 自動重連 | 連線中斷時自動嘗試恢復 |

## Slide 5b — 視訊教室：亮點

| 亮點 | 說明 |
|------|------|
| 🚀 零安裝，開瀏覽器就能上課 | 基於 WebRTC 標準，不需額外下載軟體或外掛，Chrome / Edge / Safari 皆可使用 |
| 🔒 進入前設備預檢 | 上課前即可預覽攝影機與麥克風狀態，避免課堂中才發現設備異常 |
| 📡 點對點直連，低延遲 | 音視訊資料在學生與老師瀏覽器間直接傳輸，不經過伺服器，延遲更低、體驗更流暢 |
| 🔄 自動降級與重連 | 攝影機無法開啟時自動切為純音訊模式；連線中斷自動重試最多 3 次 |
| 🌐 跨網路環境支援 | 內建 STUN/TURN 伺服器，即使在嚴格防火牆或 NAT 環境下也能成功建立連線 |
| 🖥️ 螢幕分享即開即用 | 一鍵分享整個螢幕或特定視窗，結束後自動切回攝影機畫面 |
| 💬 視訊 + 聊天一體化 | 上課期間右側面板可同步文字溝通與檔案傳送，無需切換頁面 |
| 📱 響應式設計 | 桌面、平板、手機皆可使用，自動調整介面佈局 |

## Slide 6 — 聊天室：使用者體驗流程

```
  ① 訊息中心          ② 對話列表           ③ 聊天畫面
  從側邊欄進入     →  瀏覽所有對話      →  查看歷史訊息
                      搜尋老師/學生          傳送文字或檔案
                      查看未讀數             即時收到回覆
```

**重點說明：**
- 學生端對話依「老師」分組，老師端對話依「學生」分組
- 課前課後都可使用，不限於上課時間
- 支援手機版三欄切換顯示

## Slide 7 — 聊天室：核心功能

| 功能 | 說明 |
|------|------|
| ✅ 即時文字訊息 | 即時收發，無需重新整理頁面 |
| ✅ 多媒體上傳 | 圖片、影片、音訊、文件（單檔上限 100MB） |
| ✅ 檔案下載 | 可下載對方傳送的任何類型檔案 |
| ✅ 對話搜尋與篩選 | 依名稱或科目快速找到對話 |
| ✅ 未讀訊息提示 | 對話列表顯示未讀訊息數量 |
| ✅ 連結預覽 | 自動擷取連結標題與預覽圖片 |
| ✅ 6 種訊息類型 | 文字、貼圖、語音、圖片、影片、文件 |

**支援檔案格式：**
- 圖片：.jpg / .png / .gif / .webp
- 影片：.mp4 / .webm / .mov
- 音訊：.mp3 / .wav / .ogg
- 文件：.pdf / .doc / .xls / .ppt / .txt

## Slide 7b — 聊天室：亮點

| 亮點 | 說明 |
|------|------|
| 💬 即時推送，無需重整 | 訊息透過 WebSocket 即時送達，打開頁面就能即時收到新訊息 |
| 📎 豐富多媒體支援 | 支援 6 種訊息類型（文字、貼圖、語音、圖片、影片、文件），單檔最大 100MB |
| 🔗 連結自動預覽 | 貼上網址後自動擷取標題、描述與圖片，不需手動說明連結內容 |
| 🔍 快速搜尋對話 | 依老師/學生名稱或科目關鍵字即時篩選，快速找到目標對話 |
| 🔔 未讀訊息提示 | 對話列表即時顯示未讀數量，不遺漏任何訊息 |
| 🕐 不限上課時間 | 獨立訊息中心可在課前、課後、任何時間使用，溝通不受課堂限制 |
| 👥 智慧分組對話 | 學生端依「老師」自動分組，老師端依「學生」自動分組，一目了然 |
| 📱 手機友善介面 | 三欄式佈局在手機上自動切換為單欄，支援返回手勢操作 |
| 🛡️ 雙通道保障 | WebSocket 為主、REST API 為備援，即使即時通道中斷仍可正常收發訊息 |

**關鍵說明：**

| 項目 | 說明 |
|------|------|
| 視訊連線 | 學生與老師瀏覽器透過 WebRTC 建立**點對點直連**，音視訊資料**不經過伺服器** |
| 信令交換 | 建立視訊前，雙方需透過伺服器交換連線資訊（如同「交換電話號碼」才能通話） |
| NAT 穿透 | 使用 STUN 伺服器偵測網路位址 + TURN 伺服器在無法直連時中繼傳輸 |
| 聊天訊息 | 透過伺服器中轉並**永久儲存**至資料庫 |
| 房間狀態 | 僅存伺服器記憶體，用於追蹤誰在線上 |

## Slide 8b — 系統架構概覽（聊天室 — 獨立訊息中心）

```
┌──────────────────┐                    ┌──────────────────┐
│  學生訊息中心     │                    │  老師訊息中心     │
│  StudentChat.html │                    │ teacher-messages  │
│  StudentChat.js   │                    │  TeacherChat.js   │
└────────┬─────────┘                    └────────┬─────────┘
         │                                       │
         │  REST API (歷史訊息/檔案上傳下載)       │
         │  WebSocket (即時訊息推送)               │
         ▼                                       ▼
┌──────────────────────────────────────────────────────────┐
│  ChatMessageController (REST API)                        │
│  • GET  /conversations            → 學生對話列表         │
│  • GET  /conversations/tutor/{id} → 老師對話列表         │
│  • GET  /booking/{id}             → 歷史訊息             │
│  • POST /upload                   → 上傳檔案             │
│  • GET  /download/{file}          → 下載檔案             │
│  • GET  /linkPreview?url=         → 連結預覽             │
└──────────────────────────────────────────────────────────┘
```

**兩種聊天場景的差異：**

| 比較項目 | 獨立訊息中心 | 視訊教室內嵌聊天 |
|---------|-------------|-----------------|
| 主要通訊方式 | REST API 為主，WebSocket 為輔 | WebSocket 為主，REST 為備援 |
| 使用時機 | 課前、課後、任何時間 | 上課期間 |
| 對話分組 | 學生依老師分組 / 老師依學生分組 | 依單一預約（bookingId）|
| 歷史訊息 | 可瀏覽所有對話歷史 | 僅顯示該次課程的聊天記錄 |

## Slide 9 — 角色與權限

| 項目 | 學生 | 老師 |
|------|------|------|
| 視訊教室 | ✅ 可進入 | ✅ 可進入 |
| 麥克風/攝影機 | ✅ 可控制 | ✅ 可控制 |
| 螢幕分享 | ✅ 可使用 | ✅ 可使用 |
| 訊息中心 | ✅ 對話依老師分組 | ✅ 對話依學生分組 |
| 傳送訊息/檔案 | ✅ | ✅ |
| 身份識別 | JWT Token 登入 | JWT Token 登入 |
| 頁面導向 | 學生專屬頁面 | 老師專屬頁面 |

- 登入後系統自動以 JWT Token 識別身份與角色
- 頁面載入時驗證角色，未授權者自動導向正確頁面

## Slide 10 — 資料儲存

| 資料類型 | 儲存方式 | 持久性 | 說明 |
|---------|---------|-------|------|
| 聊天記錄 | MySQL 資料庫 | ✅ 永久保存 | 所有文字與媒體訊息記錄 |

## Slide 11 — 安全機制

| 安全措施 | 防護目標 |
|---------|---------|
| JWT Token 驗證 | 確保只有已登入使用者能使用系統功能 |
| 角色權限控管 | 防止學生存取老師頁面，反之亦然 |
| XSS 防護 | 防止惡意腳本透過聊天訊息注入 |
| 檔案存取安全 | 防止未授權存取伺服器上的其他檔案 |
| 內網請求封鎖 | 連結預覽功能封鎖對內網位址的請求 |

## Slide 12 — 已知限制

| 限制 | 影響 | 風險等級 |
|------|------|---------|
| 僅支援一對一視訊 | 無法進行小組課或多人教學 | 低（目前業務僅需一對一） |
| TURN 伺服器使用公共免費服務 | 可能影響特殊網路環境下的連線穩定度 | 中 |
| 檔案存於本地 | 伺服器磁碟空間有限，無備援機制 | 中 |
| 房間狀態不持久化 | 伺服器重啟後需重新進入教室 | 低 |
| CORS 允許所有來源 | 正式環境有安全風險 | 中 |
| 無通話錄影 | 無法事後回放課程內容 | 低（視業務需求） |

## Slide 13 — 後續可優化方向

| 優化項目 | 預期效益 | 優先級建議 |
|---------|---------|-----------|
| 升級至自建/付費 TURN 伺服器 | 提升視訊連線穩定度 | 🔴 高 |
| 導入雲端物件儲存（如 AWS S3） | 解決本地磁碟空間限制，增加備援 | 🔴 高 |
| 限縮 CORS 設定 | 提升 API 安全性 | 🔴 高 |
| 加入已讀回條功能 | 提升溝通效率與使用體驗 | 🟡 中 |
| 加入推播通知 | 使用者不在線時也能收到訊息提醒 | 🟡 中 |
| 評估多人視訊需求 | 支援小組課或多人教學場景 | 🟢 低（視業務需求） |
| 課堂錄影功能 | 支援課後回放 | 🟢 低（視業務需求） |
